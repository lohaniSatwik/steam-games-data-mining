1 Business Understanding

Central question: What measurable game attributes predict commercial success on Steam, and can games be grouped into meaningful market segments to inform publishing and pricing strategy?

Supervised Tasks
  • Classification: predict whether a game receives a positive review outcome (binary: positive/negative)
  • Regression: estimate peak concurrent users (CCU) as a proxy for commercial reach
Unsupervised Task
  • Clustering: discover natural game market segments based on price, playtime, genre, and review patterns
  • Results inform positioning decisions for new releases
2 Dataset & Exploratory Data Analysis
Dataset Facts
  • ~120,000 Steam game entries
  • Features: price, genres, tags, reviews, playtime, CCU, release date, developers
  • Owner ranges stored as strings (require parsing)
  • Multi-value fields: genres, categories, tags (JSON lists)
EDA Analyses
  • Price vs. review score scatter
  • Genre & tag frequency distributions
  • Playtime and CCU outlier detection
  • Free-to-play vs. paid breakdown
  • Review score distribution & class balance
  • Correlation heatmap (numeric features)
Key Observations Expected
  • Strong class imbalance (many games have no reviews)
  • CCU highly right-skewed → log-transform
  • Price near-zero spike (free-to-play majority)
  • High cardinality in tags/genres
3 Data Preprocessing & Feature Engineering
Cleaning Steps
  • Remove games with zero reviews (no ground truth)
  • Parse estimated owner ranges → midpoint numeric value
  • Explode multi-value genre/tag lists → binary encoding (MLB)
  • Handle missing prices (DLC, bundles) → median imputation or flag column
  • Cap extreme CCU outliers (Dota 2, CS:GO) → flag as separate category
Feature Engineering
  • Log-transform CCU and price for regression
  • Derive review_ratio = positive / total reviews
  • Binning release year into era buckets
  • Normalize numeric features (StandardScaler)
  • Top-N tag/genre selection to control dimensionality
  • Export cleaned data to Parquet for reproducibility
4 Classification — Predicting Review Outcome

Target variable: Binary label derived from review_ratio — Positive (≥70% positive reviews) vs. Negative (<70%) · Supervised Binary

Baseline Required
  • Majority class classifier — always predicts the dominant class
  • Establishes minimum performance floor
Symbolic Model
  • Logistic Regression (L2 regularization)
  • Interpretable coefficients
  • Fast, good baseline for imbalanced data
Non-Symbolic Models
  • Random Forest (feature importance)
  • XGBoost (gradient boosting)
  • Handle non-linear interactions

Metrics: F1-score (macro), AUC-ROC · Class imbalance: SMOTE oversampling or class_weight='balanced'

5 Regression — Estimating Peak Concurrent Users

Target variable: peak_ccu (log-transformed) · Supervised Continuous

Baseline Required
  • Mean value regressor — always predicts the training mean
  • Provides minimum RMSE benchmark
Symbolic Model
  • Ridge Regression (L2) and Lasso (L1, implicit feature selection)
  • Coefficient magnitudes interpretable
Non-Symbolic Model
  • Gradient Boosting Regressor
  • Captures non-linear genre/tag effects on popularity

Metrics: RMSE, MAE, R² · Back-transform predictions to original scale for interpretability

6 Clustering — Game Market Segmentation

Goal: Discover natural groupings of games based on price, genre, playtime, and review patterns · Unsupervised

Methods
  • K-Means (MiniBatch variant for scale) — partition-based
  • DBSCAN — density-based, handles noise/outlier games
  • Elbow method + silhouette analysis to choose k
Evaluation & Interpretation
  • Silhouette score, Davies-Bouldin index
  • Cluster profiling: mean price, top genres, avg CCU per cluster
  • Visualize with PCA / t-SNE 2D projection
  • Assign descriptive segment labels (e.g., "AAA Action", "Indie Niche")
7 Evaluation Setup — CV, Tuning & State-of-the-Art Comparison
Cross-Validation
  • 10-fold stratified CV for all supervised models
  • Stratification preserves class ratio across folds
  • Report mean ± std of all metrics
Hyperparameter Tuning
  • Nested cross-validation (outer: 5-fold, inner: 3-fold grid search)
  • RandomizedSearchCV for XGBoost / Gradient Boost
  • Tuned params: depth, n_estimators, learning rate, regularization strength
State-of-the-Art Comparison Kaggle Required
  • Compare F1 / RMSE to top Kaggle kernels and public notebooks for this dataset
  • Cite 2–3 reference solutions from Kaggle discussions
  • Discuss where our approach matches, exceeds, or falls short
8 Error Analysis & Result Discussion
Error Analysis
  • Confusion matrix analysis for classification
  • Residual plots for regression (predicted vs. actual)
  • Identify systematic error patterns (e.g., model fails on niche genres)
  • Feature importance across all models (SHAP values for top models)
Discussion Points
  • Class imbalance impact on classification results
  • Owner range ambiguity as a limitation
  • Survivorship bias in the Steam dataset
  • Cross-task consistency: do the same features matter across tasks?
9 Conclusion & Future Work
Expected Key Findings
  • Genre and price are likely dominant predictors for review outcome
  • Log-CCU regression will reveal a long-tail commercial distribution
  • Clustering expected to recover price-tier and genre-based segments
Future Work
  • Incorporate NLP on game descriptions (TF-IDF / BERT embeddings)
  • Temporal analysis: how review scores evolve post-launch
  • Collaborative filtering → game recommender system
